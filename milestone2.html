Introduction

This is the second milestone of the project. This is a documentation of how we improved the visualization that we created in milestone 1.

Dataset

Data we are visualizing is the equipment losses on the Russia-Ukraine War. The data is obtained from a public google datasheet that is updated daily.

Original Visualization

Improved Visualization

Based on the feedbacks that we gathered on the improved visualization, we came up with our design decisions for the basis of our new visualization.

Design Decisions

  • Color Conundrum:
    • Clear indication of color where red is Russia and blue is Ukraine
  • Sniff out the important problems:
    • The amount of loss per unit
    • Interaction engaged with the information shown

Through this documentation, we will talk about the approaches made and challenges faced while doing so.

How (Idiom) is designed

  • Tooltip is used for interaction and display information of the user’s desire
  • Filtering allows users to focus on the overall or specific output

Pitfalls Encountered

Data Representation

The improved visualization was a bar chart that showed a ratio in the equipment losses between Russia and Ukraine in percentage. However, we felt that placing the actual unit loss value in each bars in the bar chart made the overall visualization confusing.

Chart Implementation

While trying to figure a way to improve upon the bar chart, we realized there were better chart implementations that were more intuitive in visualizing the data. We also wanted to make the visualization more interactive and allow the user to interact with the data.

Hence, we came up with a proposed list of design principles for our new visualization.

Design Principles
  • Emphasise the Key Areas:
    • Draws the user’s attention to the right data points by carefully choosing the size and colours
  • Smart Use of Patterns:
    • Similar forms of information can be shown as one cohesive unit across the page by using patterns
  • Proportion:
    • Indicate the weight of different data sets and the relationship between their values

Technologies Used

We implemented the chart and created the data visualisation components using HTML, JS w/ d3 and CSS. We used HTML and CSS to create and describe the elements in the visualisation. We made the data interactive using our own implementation of transformation and transition functions.
